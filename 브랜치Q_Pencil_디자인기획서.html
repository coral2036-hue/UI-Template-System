프로젝트 개요

작업 목적
HTML 프로토타입(통합_시연용_v9.html)을 기반으로 초급 React 개발자가 해당 페이지의 내용만으로 개발 및 디버깅 가능한 수준의 디자인 기획서를 Pencil.dev(.pen) 파일로 작성했습니다.

각 화면의 모든 구성요소를 디자인하고, 화면 우측에 상세 디스크립션을 배치하여 개발 인계가 가능하도록 구성했습니다.
작업 범위
해상도: 1440x900 (데스크탑 기준)
화면 구성: 풀스크린 3개 + 모달 7개 + 공통 UI 1개
디스크립션: 각 화면별 우측 DESC 프레임 6개
색상 변수: HTML :root CSS 변수 기반 23개 등록
대상 파일: 클로드코드작업 (.pen)

Pencil.dev 작업 방법

Claude Code의 Pencil MCP 도구를 활용하여 .pen 파일에 직접 디자인을 생성했습니다.

작업 방식
batch_design 명령으로 Insert/Update/Delete 연산을 일괄 실행
batch_get으로 노드 구조 조회
get_screenshot으로 시각 검증
snapshot_layout으로 겹침 검사
set_variables로 색상 토큰 등록

Pencil.dev의 flexbox 레이아웃(fill_container, fit_content)을 적극 활용하여 Auto Layout 기반 디자인 구현
Figma 대비 장점
Auto Layout 완전 지원: flexbox(horizontal/vertical) + fill_container/fit_content로 겹침 문제 근본 해결

텍스트 자동 높이 계산: textGrowth 속성으로 텍스트 크기 자동 계산

변수 시스템: $accent, $gray-200 등 CSS 변수 형태로 색상 일괄 관리

겹침 0건 달성: snapshot_layout 검증 결과 "No layout problems"

토큰 소모 분석

Pencil.dev MCP 기반 작업의 실제 토큰 소모 구조를 분석합니다. Claude Code(Opus 4.6, 1M context) 환경 기준.

토큰 소모 - 실제 과금 구조 이해 (중요)
콘텐츠 기준 (순수 생성량)
"이번 대화에서 생성한 텍스트/코드의 총량"
batch_design 코드 + 디스크립션 텍스트 + 검증 코드

이건 실제 청구와 무관한 수치입니다.
실제 과금 구조 (누적 context)
매 턴마다 대화 전체 기록이 input으로 전송됨
+ Pencil MCP 스키마/가이드(~15,000 토큰)가 매번 포함

턴이 쌓일수록 input 토큰이 눈덩이처럼 증가
Pencil MCP 특성: batch_design 1회 호출 시 Pencil 스키마(~8,000 토큰) + 가이드라인(~7,000 토큰)이 시스템 프롬프트에 포함됩니다. 이 오버헤드가 매 턴마다 반복되므로, 동일한 작업이라도 Figma Plugin API 대비 턴당 고정 비용이 높습니다.
