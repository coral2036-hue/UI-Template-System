## 3. 컴포넌트 정의

### 3-1. Input (텍스트 입력)
- 높이: **28px**
- 라벨: 좌측 고정, 입력 영역은 라벨 우측 배치
- 모든 행은 동일 baseline 유지
- 필수값: 라벨 앞 `*` 표시

### 3-2. Select (선택박스)
- 높이: **28px**, 드롭다운 화살표 아이콘 포함
- 라벨/입력 배치 규칙은 Input과 동일

### 3-3. DatePicker (날짜 입력)
- 높이: **28px**, 캘린더 아이콘 포함
- 기간 선택 시: DatePicker ~ DatePicker 형태로 구성

### 3-4. Button

| 유형 | 스타일 | 사용 시점 |
|------|--------|-----------|
| 강조 버튼 | 파란색 솔리드 | 기획서에서 강조 칠해진 버튼 |
| 라인 버튼 | 파란색 라인(외곽선) | 기획서에서 라인버튼으로 표시된 버튼 |

- 높이: **28px**, 위치: **우측 하단 정렬**

### 3-5. Grid (목록)
- 상단 헤더 + 데이터 행 구분
- 열 단위 정렬 (숫자: 우측, 텍스트: 좌측)
- 행/열 구분선 유지, 헤더 배경색 구분

### 3-6. Popup (팝업)

| 유형 | 가로폭 | 용도 |
|------|--------|------|
| 소형 (S) | 고정폭 S | 확인/취소 단순 알림 |
| 중소형 (M) | 고정폭 M | 단일 입력 폼 |
| 중형 (L) | 고정폭 L | 다중 입력 폼 |
| 대형 (XL) | 고정폭 XL | 복잡한 조회/입력 |

- 구조: **제목 영역 / 본문 영역 / 하단 버튼 영역**
- 하단 버튼: 우측 정렬, 강조+라인 버튼 조합
- 가로폭 임의 변경 금지

#### 공통 안내/고지 팝업 (WIN-CMS 전용)
- 제목: 상단 좌측 표시 + 우측 상단 X 닫기 버튼
- 본문: 섹션별 소제목(굵음) + 불릿 리스트 구성, 스크롤 지원

### 3-7. 공통 메시지 패턴
- 성공/오류/경고/안내 4종 유형
- 위치: 화면 상단 또는 인라인
- 아이콘 + 텍스트 조합

---

## 4. Form 배치 규칙

- 라벨 + 입력 행 간 **일정 간격** 유지
- 그룹 간에는 추가 간격 적용
- 동일 행 내 요소는 **baseline 통일**
- 과도한 카드 UI, 그림자, radius 금지
- spacing 수치 임의 추가 금지

## 5. Figma 프롬프팅 가이드

### 기본 프롬프트 구조

```
[화면 유형]: (메인 / 알리미 / 스케줄러 / 조회화면 / 등록화면 / 팝업 등)
[포함 컴포넌트]: (Input / Select / DatePicker / Button / Grid / Popup / 공통메시지)
[필수값 항목]: (필수값 라벨 목록)
[버튼 구성]: (강조버튼명 / 라인버튼명)
[특이사항]: (그리드 컬럼 수, 팝업 크기 유형, User Flow 참조 여부 등)
```

### 예시 1 — 조회 화면
```
다음 조건으로 WIN-CMS 업무형 시스템 조회화면 UI를 생성해줘.
- 디자인 기준: 맑은 고딕 12px, 입력 높이 28px
- 조회 조건: * 사업부(Select), * 기간(DatePicker~DatePicker), 품목명(Input)
- 버튼: [초기화](라인), [조회](강조 솔리드) — 우측 하단 정렬
- 그리드: 번호 / 사업부 / 품목명 / 수량 / 금액 / 등록일 (6컬럼)
- 스타일: 과도한 카드 UI 금지, 그림자 금지, radius 최소화
```

### 예시 2 — 등록 팝업
```
다음 조건으로 WIN-CMS 등록 팝업 UI를 생성해줘.
- 팝업 크기: 중형(L)
- 제목: 거래처 등록
- 입력항목: * 거래처명(Input), * 사업자번호(Input), 대표자명(Input), 업태(Select), 종목(Input)
- 버튼: [취소](라인), [저장](강조 솔리드) — 우측 하단 정렬
- 라벨 앞 필수값 * 표시
```

### 예시 3 — 공통 안내 팝업
```
다음 조건으로 WIN-CMS 공통 안내 팝업 UI를 생성해줘.
- 팝업 크기: 대형(XL)
- 제목: [안내 제목]
- 본문: 섹션별 소제목 + 불릿 리스트, 스크롤 가능
- 우측 상단 X 닫기 버튼 포함

```

### 예시 4 — 메인 대시보드
```
다음 조건으로 WIN-CMS 메인 대시보드 UI를 생성해줘.
- 상단: 공통 네비게이션 헤더
- 본문: 알리미 요약 / 스케줄러 요약 / 통계 위젯 배치
- 강조 색상: 파란색 (#1659C5), 서브 강조: 주황색
- 스타일: 카드 UI 최소화, 그림자 금지
```

### 금지 사항
```
- 신규 스타일 생성 금지
- hover, disabled 상태 생성 금지
- spacing 수치 임의 추가 금지
- 팝업 폭 수치 임의 추가 금지
- 과도한 카드 UI / 그림자 / radius 금지
- 시스템명(WIN-CMS) 외 브랜드 스타일 적용 금지
```
