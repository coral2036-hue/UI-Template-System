## 1. 사용 의도

엑셀 분석기는 **상세조회 페이지에서 필터링된 데이터를 보고서빌더(B1)에서 MD/JSON으로 편집하여 그래프+테이블+통계를 조합한 분석 보고서를 만드는** 기능이다. 엑셀 편집기(사이드패널)와는 별도의 전용 페이지이다.

| 사용 동기 | 예시 |
|----------|------|
| 데이터 시각화 | 필터링된 자금 데이터를 차트+통계 카드로 변환 |
| 보고서 작성 | MD 편집으로 제목/본문/테이블/차트를 조합한 정형 보고서 생성 |
| 템플릿 활용 | 기본 템플릿으로 빠르게 보고서 구조 생성 |
| 외부 파일 활용 | .md/.txt 파일 가져와서 보고서 내용 채우기 |
| 보고서 저장/배포 | 완성된 보고서를 저장 → PDF/공유/이메일 배포 |

## 2. 진입 경로

```mermaid
flowchart LR
    AI["AI 챗봇 응답<br/>(data-table 블록)"] -- 상세조회 --> DETAIL["상세조회 페이지<br/>(필터+테이블+페이지네이션)"]

    DETAIL -- "[데이터분석] 클릭<br/>openReportBuilder(bid)" --> BUILDER["보고서빌더 (B1)<br/>/builder 페이지"]

    DETAIL -- "[Excel] 클릭<br/>urExcel(bid)" --> EXCEL["엑셀 편집기<br/>(사이드패널)"]

    SB["사이드바<br/>맞춤보고서"] -- 직접 진입 --> BUILDER
```

**두 가지 진입 경로:**
1. 상세조회 페이지 → [데이터분석] 버튼 → `openReportBuilder(bid)` → 보고서빌더 오픈
2. 사이드바 → 맞춤보고서 → 보고서빌더 직접 진입

## 3. 메인 업무흐름도

```mermaid
flowchart TD
    START([필터링된 데이터를<br/>분석 보고서로 만들고 싶다]) --> OPEN

    OPEN["보고서빌더(B1) 진입<br/>Route: /builder"] --> LAYOUT

    LAYOUT["화면 구성:<br/>Sidebar(70) + BuilderHeader(52)<br/>+ 좌:프리뷰(960) + 우:에디터(410)<br/>+ BottomBar(52)"]

    LAYOUT --> STATE{현재 상태}

    STATE -- 빈 상태 --> EMPTY["프리뷰: 'MD 편집 후<br/>프리뷰 적용 버튼을 클릭하세요'<br/>에디터: 빈 입력창"]
    STATE -- 입력 상태 --> FILLED["프리뷰: 블록 렌더링 결과<br/>에디터: MD/JSON 내용"]

    EMPTY --> EDIT_START{편집 시작 방법}

    EDIT_START -- 직접 입력 --> MD_EDIT["MD/JSON 에디터에<br/>마크다운 직접 입력"]
    EDIT_START -- 파일 가져오기 --> IMPORT["📥 파일 가져오기 클릭<br/>.md/.txt 파일 선택<br/>→ 에디터에 내용 로드"]
    EDIT_START -- 기본 템플릿 --> TEMPLATE["기본 템플릿 클릭<br/>→ 사전정의 마크다운 자동 로드"]

    MD_EDIT --> PREVIEW
    IMPORT --> PREVIEW
    TEMPLATE --> PREVIEW

    PREVIEW["👁 프리뷰 적용 클릭<br/>→ 좌측에 블록 렌더링<br/>(number-stat, data-table,<br/>bar-chart 등)"]

    PREVIEW --> CHECK{결과 확인}
    CHECK -- 수정 필요 --> MD_EDIT
    CHECK -- 완성 --> SAVE

    SAVE["💾 저장 클릭<br/>→ B2-보고서저장모달 표시"]
    SAVE --> SAVE_OK{저장 성공?}
    SAVE_OK -- 성공 --> DONE["토스트: 저장 완료<br/>→ 리포트 목록에 추가"]
    SAVE_OK -- 실패 --> SAVE_ERR["토스트: 저장 실패<br/>재시도 가능"]

    DONE --> AFTER{후속 행동}
    AFTER -- PDF 다운로드 --> PDF["보고서 PDF 변환 + 다운로드"]
    AFTER -- 공유 --> SHARE["공유 모달(M3)"]
    AFTER -- 추가 편집 --> MD_EDIT
    AFTER -- 완료 --> END([분석 완료])
```


| 상태 | 에디터 | 프리뷰 |
|------|-------|--------|
| 빈 상태 | 빈 입력창 | "MD 편집 후 프리뷰 적용 버튼을 클릭하세요" |
| 입력 상태 | MD/JSON 텍스트 | 블록 렌더링 결과 (number-stat, data-table 등) |

## 5. 상세조회 → 분석기 연동 흐름

```mermaid
flowchart TD
    DT["상세조회 페이지에서<br/>기간 필터링 완료<br/>(filteredRows 확보)"] --> BTN["[데이터분석] 버튼 클릭"]

    BTN --> PASS["openReportBuilder(bid)<br/>filteredRows 데이터를<br/>보고서빌더에 전달"]

    PASS --> BUILDER["보고서빌더 오픈<br/>전달받은 데이터 기반으로<br/>MD 템플릿 자동 생성"]

    BUILDER --> AUTO["자동 생성 내용:<br/>- 제목 + 기준일<br/>- number-stat (합계/평균/건수)<br/>- data-table (filteredRows)<br/>- 차트 데이터"]

    AUTO --> EDIT["사용자가 MD 수정<br/>제목 변경, 항목 추가/삭제 등"]

    EDIT --> APPLY["프리뷰 적용 → 결과 확인"]
    APPLY --> SAVE_FLOW["저장 → PDF/공유"]
```

## 6. 에러 처리

| 상황 | 처리 | 사용자 피드백 |
|------|------|-------------|
| 파일 가져오기 형식 오류 | .md/.txt 외 파일 차단 | "지원하지 않는 파일 형식입니다" |
| MD 파싱 실패 | 프리뷰에 원본 텍스트 표시 | 토스트: "마크다운 변환 실패" |
| 프리뷰 렌더링 실패 | 에러 블록만 스킵 | 정상 블록은 표시 |
| 저장 실패 | 재시도 가능 | 토스트: "저장 실패" |
| 데이터 전달 실패 | 빈 상태로 빌더 오픈 | 직접 입력 안내 |

## 7. 업무 시나리오

### A. 상세조회 → 분석 보고서 생성
```
AI "법인카드 현황" → 상세조회 → 2월 필터링 → [데이터분석]
→ 보고서빌더에 데이터 자동 로드 → MD 수정 → 프리뷰 → 저장 → PDF 다운
```

### B. 템플릿으로 정기 보고서
```
사이드바 "맞춤보고서" → 보고서빌더 → 기본 템플릿 로드
→ 데이터 수정 → 프리뷰 적용 → 저장
```

### C. 외부 파일 기반 보고서
```
보고서빌더 → 📥 파일 가져오기 → 기존 .md 파일 로드
→ 수정/보완 → 프리뷰 → 저장 → 이메일 공유
```
