# BranchQ E2E 테스트 시나리오

## 개요
시나리오 기반의 End-to-End 테스트. 실제 사용자 흐름을 따라 전체 기능을 검증합니다.

---

## 시나리오 1: 첫 방문 → 추천 질문 → 블록 확인

**목적**: 신규 사용자가 처음 접속하여 추천 질문으로 대화를 시작하는 흐름

**사전 조건**: 앱 첫 로드 (S0 Welcome 화면)

| # | 액션 | 기대 결과 |
|---|------|----------|
| 1 | 페이지 로드 | "🏦 BranchQ" + "무엇을 도와드릴까요?" 표시 |
| 2 | 카테고리 필 확인 | 5개 필 표시, "일반질의" 활성 (파란색) |
| 3 | 추천 질문 확인 | 일반질의 관련 3개 질문 카드 표시 |
| 4 | "이번 주 법인카드 사용 내역을 조회해줘" 클릭 | User 메시지 버블 표시 (우측 파란색) |
| 5 | 로딩 | ●●● 타이핑 인디케이터 0.8~1.6초 |
| 6 | AI 응답 확인 | TextContent + DataTable + RelatedQuestions 블록 |
| 7 | DataTable 검증 | 헤더: 거래일/가맹점/금액/상태, 5행 데이터 |
| 8 | 금액 컬럼 | 우측정렬, Inter Bold 폰트 |
| 9 | 배지 컬럼 | "승인"(초록), "검토중"(노랑) |
| 10 | ShareRow | "PDF" + "공유" 버튼 표시 |
| 11 | 카테고리 필 바 | ChatArea 상단에 5개 필 고정 표시 |

## 시나리오 2: 키워드 매칭 검증

**목적**: 직접 입력한 텍스트가 올바른 씬으로 매칭되는지 확인

**사전 조건**: S0 Welcome 화면

| # | 입력 텍스트 | 기대 매칭 | 검증 블록 |
|---|------------|----------|----------|
| 1 | "카드 사용 내역 조회" | general (S1) | DataTable (법인카드 사용 내역) |
| 2 | "법인카드 분석 리포트" | analysis (S2) | ReportHeader + NumberStat(4개) + PatternAnalysis |
| 3 | "자금흐름 예측" | forecast (S3) | DataTable + Callout + BarChart(그룹) |
| 4 | "이상거래 현황" | anomaly (S4) | DataTable (배지+점수), 점수색상 검증 |
| 5 | "계좌 등록 방법" | consult (S5) | Steps(4단계) + SourceBox(다운로드) |
| 6 | "1분기 보고서 작성" | report (S6) | ApprovalBox + NumberStat + BarChart |
| 7 | "안녕하세요" (매칭 없음) | general (기본) | DataTable (기본 응답) |

**검증 방법**: 각 텍스트를 InputBar에 입력 → Enter → AI 응답 블록 확인

## 시나리오 3: 다중 턴 대화 + 카테고리 전환

**목적**: 여러 질문을 연속으로 하면서 대화가 유지되는지, 카테고리 전환이 정상 동작하는지

| # | 액션 | 기대 결과 |
|---|------|----------|
| 1 | "카드 내역 조회" 입력 | S1 응답, 대화 1턴 |
| 2 | "이상거래 확인해줘" 입력 | S4 응답, 이전 대화 유지 (총 2턴) |
| 3 | 스크롤 확인 | 4개 메시지 (User1, AI1, User2, AI2) 순서대로 |
| 4 | 카테고리 필에서 "예측" 클릭 | 새 대화로 전환 (S3 씬 로드) |
| 5 | "새 채팅" 클릭 (사이드바) | 메시지 초기화 → S0 Welcome 복귀 |

---

## 시나리오 4: 모달 플로우 (공유 → 이메일 → 토스트)

**목적**: 모달 연쇄 동작과 토스트 알림 검증

**사전 조건**: 대화 중 AI 응답이 표시된 상태

| # | 액션 | 기대 결과 |
|---|------|----------|
| 1 | ShareRow "공유" 클릭 | M3 공유 모달 열림 (4개 옵션) |
| 2 | "이메일" 옵션 클릭 | M3 닫힘 → M5 이메일 모달 열림 |
| 3 | 받는 사람 비어있는 상태 | "전송" 버튼 비활성 (회색) |
| 4 | 받는 사람 입력 | "전송" 버튼 활성 (파란색) |
| 5 | "전송" 클릭 | 모달 닫힘 + 토스트 "이메일이 전송되었습니다." |
| 6 | 토스트 자동 소멸 | 3초 후 자동으로 사라짐 |
| 7 | GPT-4 버튼 클릭 | M1 모델 선택 모달 열림 |
| 8 | "Claude 3.5" 선택 → "적용" | 모달 닫힘 + 헤더 모델명 "Claude 3.5"로 변경 |
| 9 | ESC 키 테스트 | 아무 모달 열기 → ESC → 닫힘 |
| 10 | 외부 클릭 테스트 | 아무 모달 열기 → 오버레이 클릭 → 닫힘 |

## 시나리오 5: 반응형 테스트

**목적**: Desktop → Tablet → Mobile 뷰포트 전환 시 레이아웃 검증

### Desktop (1280px+)
| # | 검증 항목 | 기대 결과 |
|---|----------|----------|
| 1 | 사이드바 | 70px, 아이콘+라벨 표시 |
| 2 | 헤더 | BranchQ + 모델 버튼 + 도움말/알림 |
| 3 | ChatArea | max-w 800px 중앙, 카테고리 필 상단 고정 |
| 4 | SubPanel | 열기 시 ChatArea 옆에 320px 슬라이드 |

### Tablet (768px)
| # | 검증 항목 | 기대 결과 |
|---|----------|----------|
| 1 | 사이드바 | 아이콘만 표시 (라벨 숨김) |
| 2 | 나머지 레이아웃 | Desktop과 동일 |

### Mobile (640px 미만)
| # | 검증 항목 | 기대 결과 |
|---|----------|----------|
| 1 | 사이드바 | 완전 숨김 |
| 2 | 햄버거 메뉴 | 헤더 좌측에 표시 |
| 3 | 햄버거 클릭 | 오버레이 + 사이드바(200px) 슬라이드 인 |
| 4 | 오버레이 클릭 | 사이드바 닫힘 |
| 5 | X 버튼 | 사이드바 닫힘 |
