6개 카테고리 탭 + 텍스트 입력 + 모델 선택기. 사용자 질문 입력의 핵심 부품.

report-header, number-stat, data-table, bar-chart, line-chart, callout, steps, source-box 등.

■ API 설정 모달

4개 Provider 탭(BranchQ/Anthropic/OpenAI/Google). API 키 입력, 상태 배지, 3단계 설정 가이드.

MMS/카카오톡/이메일 전송. 카카오톡 미리보기(스마트폰 프레임), 이메일 미리보기(macOS 스타일).

■ 보고서 빌더

MD 에디터 + 미리보기 좌우 분할. 파일 가져오기, 기본 템플릿, 글자 수 카운터.

일반질의, 분석질의, 예측질의, 이상거래, 상담, 보고서. 각각 다른 블록 조합으로 표현.

교훈 및 시사점

개발 과정에서 얻은 교훈

작은 도구에서 시작하기

v25 프롬프트 생성기라는 "도구의 도구"에서 시작. 내부 도구의 CSS 변수/패널 구조가 최종 제품의 뼈대가 되었습니다. 처음부터 완성품을 만들지 않고, 도구를 먼저 만들어 반복 효율을 높인 접근입니다.

파일 이름에 "뭘 고쳤는지" 기록

v2_input_fix, v4_column_spacing_fix 등 파일 이름 자체가 변경 이력입니다. 별도의 변경 로그 없이도 "어떤 순서로 뭘 고쳤는지" 추적이 가능합니다.

다양한 디자인을 병행 탐색

메인 개발과 별개로 뱅킹 홈/ChatGPT 스타일/Before-After 비교를 동시에 시도. 최적의 패턴을 고른 뒤 통합하는 방식이 결과물의 완성도를 높였습니다.

안정화 기간을 충분히 가지기

통합_시연용 v1~v8(3/26~4/1, 6일)은 크기가 거의 변하지 않았습니다. 기능 추가 없이 기존 코드를 다듬는 안정화 기간을 거친 뒤 v9에서 대규모 통합을 진행했습니다.

참조 구현을 먼저 만들기
