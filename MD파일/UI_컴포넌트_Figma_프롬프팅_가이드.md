# UI 컴포넌트 가이드 + Figma 프롬프팅 가이드
> 출처: 디자인 공유 커뮤니케이션용(가이드 및 확인) Figma 파일 + 공통 디자인 가이드 문서 기반

---

## 1. 공통 디자인 토큰

| 항목 | 값 |
|------|-----|
| 기본 폰트 | 맑은 고딕 |
| 기본 폰트 크기 | 12px |
| 입력/버튼 높이 | 28px |
| 강조 버튼 색상 | 파란색 솔리드 (#0066CC 계열) |
| 라인 버튼 색상 | 파란색 라인 (테두리만) |
| 필수값 표시 | * (라벨 앞) |
| 기본 정렬 | 좌측 정렬 |

---

## 2. 컴포넌트 정의

### 2-1. Input (텍스트 입력)
- 높이: **28px**
- 라벨: 좌측 고정
- 입력 영역: 라벨 우측 배치
- 모든 행은 동일 시작선(baseline) 유지
- 필수값: 라벨 앞 `*` 표시

### 2-2. Select (선택박스)
- 높이: **28px** (Input과 동일)
- 드롭다운 화살표 아이콘 포함
- 라벨/입력 배치 규칙은 Input과 동일

### 2-3. DatePicker (날짜 입력)
- 높이: **28px**
- 캘린더 아이콘 포함
- Input/Select와 동일한 높이 기준 유지

### 2-4. Button

| 유형 | 스타일 | 사용 시점 |
|------|--------|-----------|
| 강조 버튼 | 파란색 솔리드 | 기획서에서 강조 칠해진 버튼 |
| 라인 버튼 | 파란색 라인(외곽선) | 기획서에서 라인버튼으로 표시된 버튼 |

- 높이: **28px**
- 위치: **우측 하단 정렬**

### 2-5. Grid (목록)
- 상단 헤더와 데이터 행 구분
- 열 단위 정렬 유지 (숫자: 우측, 텍스트: 좌측)
- 행/열 구분선 유지
- 헤더: 굵은 텍스트, 배경색 구분

### 2-6. Popup (팝업)

| 유형 | 가로폭 | 용도 |
|------|--------|------|
| 소형 | 고정폭 S | 확인/취소 단순 알림 |
| 중소형 | 고정폭 M | 단일 입력 폼 |
| 중형 | 고정폭 L | 다중 입력 폼 |
| 대형 | 고정폭 XL | 복잡한 조회/입력 |

- 구조: **제목 영역 / 본문 영역 / 하단 버튼 영역**
- 하단 버튼: 우측 정렬, 강조+라인 버튼 조합
- 가로폭은 유형별 고정값 사용 (임의 변경 금지)

---

## 3. Form 배치 규칙

- 라벨 + 입력 행 간 **일정 간격** 유지
- 그룹 간에는 추가 간격 적용
- 동일 행 내 요소는 **baseline 통일**
- 과도한 카드 UI, 그림자, radius 금지
- spacing 수치 임의 추가 금지

## 4. Figma 프롬프팅 가이드

### 기본 프롬프트 구조

```
[화면 유형]: (조회화면 / 등록화면 / 팝업 등)
[포함 컴포넌트]: (Input / Select / DatePicker / Button / Grid / Popup)
[필수값 항목]: (필수값 라벨 목록)
[버튼 구성]: (강조버튼명 / 라인버튼명)
[특이사항]: (그리드 컬럼 수, 팝업 크기 유형 등)
```

### 예시 1 — 조회 화면

```
다음 조건으로 업무형 시스템 조회화면 UI를 생성해줘.
- 디자인 기준: 맑은 고딕 12px, 입력 높이 28px
- 조회 조건: * 사업부(Select), * 기간(DatePicker~DatePicker), 품목명(Input)
- 버튼: [초기화](라인), [조회](강조 솔리드) — 우측 하단 정렬
- 그리드: 번호 / 사업부 / 품목명 / 수량 / 금액 / 등록일 (6컬럼)
- 스타일: 과도한 카드 UI 금지, 그림자 금지, radius 최소화
```

### 예시 2 — 등록 팝업

```
다음 조건으로 업무형 시스템 등록 팝업 UI를 생성해줘.
- 팝업 크기: 중형(L)
- 제목: 거래처 등록
- 입력항목: * 거래처명(Input), * 사업자번호(Input), 대표자명(Input), 업태(Select), 종목(Input)
- 버튼: [취소](라인), [저장](강조 솔리드) — 우측 하단 정렬
- 라벨 앞 필수값 * 표시
```

### 예시 3 — 버튼 판단 기준

```
기획서에서 파란 형광펜으로 칠해진 버튼 → 파란색 솔리드 강조 버튼
기획서에서 라인만 표시된 버튼 → 파란색 라인 버튼
모든 버튼 높이 28px, 우측 하단 정렬
```

### 금지 사항

```
- 신규 스타일 생성 금지
- hover, disabled 상태 생성 금지
- spacing 수치 임의 추가 금지
- 팝업 폭 수치 임의 추가 금지
- 과도한 카드 UI / 그림자 / radius 금지
```
