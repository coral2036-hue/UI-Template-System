## 2. 공통 디자인 규칙

| 항목 | 값 |
|------|-----|
| 폰트 | 맑은 고딕 12px |
| 입력/버튼 높이 | 28px |
| 강조 버튼 | 파란색 솔리드 |
| 라인 버튼 | 파란색 라인 |
| 필수값 표시 | * (라벨 앞) |
| 헤더 강조색 | 파란색 (#1659C5) |

---

## 3. 화면별 컴포넌트 구성

### 3-1. 은행계좌통합조회
- 조회 조건: Select + DatePicker 기간 조회
- 그리드: 계좌번호 / 은행명 / 잔액 등
- 버튼: [초기화](라인) / [조회](강조)

### 3-2. 은행계좌전액조회
- 조회 조건: Select + Input 복합
- 버튼: [초기화](라인) / [조회](강조)

### 3-3. 은행계좌기거래내역조회
- 조회 조건: 계좌 Select + 기간 DatePicker
- 그리드: 거래일시 / 거래유형 / 금액 / 잔액

### 3-4. 증권계좌조회
- 조회 조건: 계좌번호 Select
- 그리드: 증권 잔고 / 평가금액

### 3-5. 기타조회
- 안내 팝업 포함 (대형 XL)
- 조회 결과 없음 상태 표시 포함

## 4. Figma 프롬프팅 예시

```
WIN-CMS 은행계좌통합조회 화면을 생성해줘.
- 조회 조건: * 은행(Select), * 기간(DatePicker~DatePicker), 계좌번호(Input)
- 버튼: [초기화](라인), [조회](강조 솔리드) — 우측 하단
- 그리드: 번호 / 은행명 / 계좌번호 / 잔액 / 거래일 (5컬럼)
- 스타일: 맑은 고딕 12px, 높이 28px, 그림자 금지
```
